# Task: Feature Implementation

## Input Required
- Feature name
- Affected domain(s)
- Acceptance criteria

## Process
1. Identify affected bounded context
2. Define domain changes (entities, events)
3. Define application changes (use cases)
4. Define infrastructure changes (repos, adapters)
5. Define API changes (endpoints, serializers)
6. List files to create/modify

## Output Format
```
## Domain Changes
[List entities, value objects, events]

## Application Changes
[List use cases, handlers]

## Infrastructure Changes
[List repositories, adapters]

## API Changes
[List endpoints, request/response]

## Files
[List with paths]

## Implementation Order
[Numbered steps]
```

## Constraints Check
Before implementation, verify against system/constraints.md.
