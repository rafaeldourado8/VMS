fix: corrige CSS do Django Admin e CSRF via Kong/HAProxy

## Problemas Corrigidos

### 1. Django Admin sem CSS
- Arquivos estáticos não eram servidos através do Kong
- Adicionada rota `/static` e `/media` no Kong
- Kong agora roteia para Nginx corretamente

### 2. Erro CSRF 403
- Django bloqueava requests via proxy
- Configurado CSRF_TRUSTED_ORIGINS para aceitar Kong/HAProxy
- Ajustadas configurações de cookies para desenvolvimento

### 3. Erro 503 no HAProxy
- Validado roteamento HAProxy → Kong → Backend
- Confirmado healthchecks funcionando

## Arquivos Modificados

### Configuração
- kong/kong.yml: Nova rota `django-static` para servir /static e /media
- backend/config/settings.py: 
  - CSRF_TRUSTED_ORIGINS expandido
  - CSRF_COOKIE_SAMESITE = 'Lax'
  - Cookies seguros desabilitados em DEBUG mode

### Scripts de Diagnóstico (Novos)
- fix-css.bat: Correção automática de CSS
- check-static.bat: Diagnóstico de arquivos estáticos
- open-admin.bat: Abre admin pela porta correta
- diagnose.bat: Diagnóstico completo do sistema

### Documentação (Nova)
- TROUBLESHOOTING-CSS.md: Guia completo de troubleshooting
- README-CSS-FIX.md: Guia rápido de uso dos scripts
